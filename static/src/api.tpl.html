API and datasets

  • API v1
  • API v2
  • Datasets

There are two ways to get access to the oaDOI dataset: the API is best for most uses because it's simple, fast, and easy to get running; the datasets are great for large-scale research and data mining. Whichever you're using, we recommend you subscribe to the mailing list in order to stay up-to-date when there are changes or new features. Keep in mind when using either the API or dataset that occasionally experimental keys/columns may appear, in addition to those described below; of course, don't build anything important on those until they're officially documented.

API version 1

This is currently the default version; however v2 will become the default on October 1, 2017, so we recommend new users build on v2.

The REST API gives anyone free, programmatic access to all of oaDOI's data. There's no rate limit, but if you need more than 100k calls/day you may want to download the datasets instead.

Requests must include your email, so that we can get in touch if something breaks, and so we can report usage to our funders. Add the email as a parameter at the end of the URL, like this: ?email=YOUR_EMAIL.

GET /

gets information about the API. Returns a status object with version number. Try it here: https://api.oadoi.org?email=test@example.com

documentation_url String Where you can find documentation for this version.
msg String Relevant messages.
version String Version string. Example: 1.3.0
GET /:doi

gets data about a single DOI. Try this example: https://api.oadoi.org/10.1038/nature12373?email=test@example.com

algorithm_version Integer Indicates the data collection approaches used for this article.

Possible values

  • 1 First-generation hybrid detection. Uses only data from the Crossref API to determine hybrid status. Does a good job for Elsevier articles and a few other publishers, but most publishers are not checked for hybrid.
  • 2 Second-generation hybrid detection. Uses additional sources, checks all publishers for hybrid. Gets about 10x as much hybrid. data_standard==2 is the version used in the paper we wrote about oaDOI.
doi String The requested DOI.
doi_resolver String The organization in charge of issuing and resolving this DOI.

Possible values

  • crossref This is a Crossref DOI
  • datacite This is a Datacite DOI.
evidence String Free-text string describing how we found the free_fulltext_url.

Used for debugging. Don’t depend on the exact contents of this for anything, because values are subject to change without warning.

Example values

  • oa journal (via journal title in doaj) We found the name of the journal that publishes this article in the DOAJ database.
  • oa repository (via pmcid lookup) We found this article in an index of PubMed Central articles.
  • ...
free_fulltext_url String URL where the best fulltext copy of this article lives.

The fulltext can be in various formats, including PDF, HTML, or even Word.

is_boai_license Boolean True whenever the license is cc-by, cc0, or PD.

This key attempts to identify articles that meets the widely-used BOAI definition of Open Access.

is_free_to_read Boolean True whenever the free_fulltext_url is not None.
is_subscription_journal Boolean True whenever the journal is not in the DOAJ or DataCite.
license String The license under which the article is published.

Example values

  • Creative Commons licenses are uniformly abbreviated and lowercased. Example: cc-by-nc
  • Publisher-specific licenses are normalized using this format: acs-specific: authorchoice/editors choice usage agreement
oa_color String Provides information about the host of the OA copy

Possible values

  • Gold means the free_fulltext_url is served by the article's publisher.
  • Green means the free_fulltext_url is served from an OA repository.
url String The URL that the DOI resolves to.

The REST API gives anyone free, programmatic access to all of oaDOI's data. There's no rate limit, but if you need more than 100k calls/day you may want to download the datasets instead.

Possible values

  • 1 First-generation hybrid detection. Uses only data from the Crossref API to determine hybrid status. Does a good job for Elsevier articles and a few other publishers, but most publishers are not checked for hybrid.
  • 2 Second-generation hybrid detection. Uses additional sources, checks all publishers for hybrid. Gets about 10x as much hybrid. data_standard==2 is the version used in the paper we wrote about oaDOI.
doi String The DOI of this article. This is always lowercase.
is_oa Boolean True if oa_url is not Null.
journal_issns String Any ISSNs assigned to the journal publishing this resource. Separate ISSNs are sometimes assigned to print and electronic versions of the same journal. If there are multiple ISSNs, they are separated by commas. Example: 1232-1203,1532-6203
journal_name String The name of the journal publishing this resource. The same journal may have multiple name strings (eg, "J. Foo", "Journal of Foo", "JOURNAL OF FOO", etc). These have not been fully normalized within our database, so use with care.
oa_archived_somewhere Boolean TODO
oa_evidence String Hows we found the oa_url.

Used for debugging. Don’t depend on the exact contents of this for anything, because values are subject to change without warning. Example values:

  • oa journal (via journal title in doaj) We found the name of the journal that publishes this article in the DOAJ database.
  • oa repository (via pmcid lookup) We found this article in an index of PubMed Central articles.
oa_host_type String The type of host that serves oa_url.

Describes the host that serves this particular copy of this article living at oa_url. There are two possible values:

  • publisher means the oa_url is served by the article’s publisher (in practice, this means it is hosted on the same domain the DOI resolves to).
  • repository means the oa_url is served by an Open Access repository, rather than by the publisher.
oa_is_doaj_journal Boolean True if the journal hosting this resource is listed in the DOAJ. It is common to define gold OA based on inclusion in the DOAJ.
oa_license String The license under which the article is published.

We return several types of licenses:

  • Creative Commons licenses are uniformly abbreviated and lowercased. Example: cc-by-nc
  • Publisher-specific licenses are normalized using this format: acs-specific: authorchoice/editors choice usage agreement
  • When we have evidence that an OA license of some kind was used, but it’s not reported directly on the article page, this field returns implied-oa
updated String Time when the data for this record was last updated. Returned as an ISO8601-formatted timestamp. Example: 2017-08-17T23:43:27.753663
oa_url String URL where the best fulltext copy of this article lives.

The fulltext can be in various formats, including PDF, HTML, or even Word.

Datasets

The datasets behind the oaDOI API are available for download as two big CSV files. They are free for use in non-commercial academic research. Contact us if you're interested in downloading it.

If you'd like to download the datasets for commercial use, check out our service-level agreement program.
